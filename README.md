# Nuke Assalt
Generated by [Rojo](https://github.com/rojo-rbx/rojo) 7.7.0-rc.1.

## Workflow
This repository is the source of truth for code and static configuration. Roblox Studio is used as the runtime and sync target in a Studio-first workflow.

## Generate Config
Generate the Luau config modules from the canonical JSON files before opening the place in Studio:

```powershell
$env:DOTNET_CLI_HOME="$PWD\\.dotnet"
$env:NUGET_PACKAGES="$PWD\\.nuget\\packages"
dotnet run --project .\tools\NukeAssalt.Tools\NukeAssalt.Tools.csproj -- generate-config
```

## Validate
Run the stage validation suite with:

```powershell
$env:DOTNET_CLI_HOME="$PWD\\.dotnet"
$env:NUGET_PACKAGES="$PWD\\.nuget\\packages"
dotnet test .\tools\NukeAssalt.Validation.sln
```

## Rojo Sync
To build the place from scratch, use:

```bash
rojo build -o "Nuke Assalt.rbxlx"
```

Next, open `Nuke Assalt.rbxlx` in Roblox Studio and start the Rojo server:

```bash
rojo serve
```

Then connect Roblox Studio to the local Rojo server and verify the tree syncs into `ReplicatedStorage`, `ServerScriptService`, and `StarterPlayerScripts`.

For more help, check out [the Rojo documentation](https://rojo.space/docs).
